

##Scenario 1: Application not accessible via Apache but working on JBoss:

Situation:
          http://server:8080/app  --> working
          http://server/app --> not working

Troubleshooting:
 
Command: systemctl status httpd

Check config:
              grep -i proxy /etc/httpd/conf/httpd.conf

Check mod_jk:
            cat /etc/httpd/conf.s/workers.properties

Check AJP port:
               netstat -tulpn | grep 8009

Check logs:
            tail -f /var/log/httpd/error_log

Root Cause:
           Proxy/mod_jk misconfiguration OR AJP not enabled

Scenario 2: 503 Service Unavailble:

Troubleshooting:

Check JBoss:
            ps -ef | grep java

Check Apache error log:
           tail -f /var/log/httpd/error_log

Possible Causes:
               JBoss down
               AJP connection failure
               Worker not reachable
